# Verb Classes (v1) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add heap-allocated, GC-managed classes (`forge`/`has`/`spawn`/`.`) to the Verb compiler, distinct from a future `struct` feature.

**Architecture:** New `TAG_OBJECT=8` heap value laid out `[refcount hdr][i64 class_id][VerbValue slots…]`, allocated through the existing `verb_alloc` and freed by the reference-counting GC. Classes are collected into a compile-time class table; field/method access sites know the *name* statically and emit a `switch` on the runtime `class_id`. Everything is generated LLVM IR — no C++ runtime, so it works under both JIT (`verb run`) and AOT (`verb build`).

**Tech Stack:** Rust 2021, `inkwell` (LLVM 20.1), existing lexer/parser/codegen in `src/`, e2e harness in `tests/e2e.rs` driving `.verb` fixtures.

## Global Constraints

- Spec: `docs/superpowers/specs/2026-07-22-classes-design.md` — authority for all behavior.
- Keywords are verbs: class = `forge`, reserved struct keyword = `shape` (parser rejects, do not implement). Fields = `has`, methods = `make`, constructor = method named `init`, instantiation = `spawn`, member access = `.`.
- `self` and `init` are **not** keywords: `self` is an ordinary parameter name; `init` is recognized only by position (a method named `init`).
- Object runtime layout is fixed: `[i64 class_id]` at offset 0, then `VerbValue` slots (16 bytes each) in field-declaration order; total alloc `8 + 16*nfields` bytes via `verb_alloc` (never `malloc`) so the GC refcount header at `ptr-8` is valid.
- `equals`/`differs` on objects = reference identity (same pointer), matching arrays.
- `print(obj)` renders `Name { field: value, ... }` in declaration order.
- No inheritance, no computed member access, no static methods in v1.
- Follow existing codegen conventions exactly: retain-on-load, release-old-before-store, ownership-transfer of call args. Mirror the cited analog code rather than inventing new patterns.
- TDD: write the failing test first, watch it fail, implement, watch it pass, commit. One logical change per commit.
- Build/test commands: `cargo build`, `cargo test`, single e2e: `cargo test --test e2e <name>`, single unit: `cargo test <name>`.

- [ ] **Step 4: Implement `forge_stmt`**

Add after `fn_stmt` (`parser.rs:249`):

```rust
    fn forge_stmt(&mut self) -> Result<Stmt, CompileError> {
        let (line, col) = self.here();
        self.advance(); // forge
        let (name, _, _) = self.expect_ident("class name after 'forge'")?;
        self.expect(&TokenKind::Begin, "'begin'")?;
        let mut fields = Vec::new();
        let mut methods = Vec::new();
        while !self.check(&TokenKind::End) && !self.check(&TokenKind::Eof) {
            match self.peek() {
                TokenKind::Has => {
                    self.advance();
                    let (f, _, _) = self.expect_ident("field name after 'has'")?;
                    if fields.contains(&f) {
                        return Err(self.err(format!("duplicate field '{f}' in class '{name}'")));
                    }
                    fields.push(f);
                    self.expect(&TokenKind::Semi, "';'")?;
                }
                TokenKind::Make => {
                    let (mline, mcol) = self.here();
                    // reuse fn_stmt then unwrap into a Method
                    match self.fn_stmt()? {
                        Stmt::Fn { name: mname, params, body, .. } => {
                            if methods.iter().any(|m: &Method| m.name == mname) {
                                return Err(self.err(format!("duplicate method '{mname}' in class '{name}'")));
                            }
                            methods.push(Method { name: mname, params, body, line: mline, col: mcol });
                        }
                        _ => unreachable!("fn_stmt always returns Stmt::Fn"),
                    }
                }
                _ => return Err(self.err("expected 'has' or 'make' in class body")),
            }
        }
        self.expect(&TokenKind::End, "'end'")?;
        Ok(Stmt::Forge { name, fields, methods, line, col })
    }
```

- [ ] **Step 6: Add `.` postfix to `call`**

Replace the `while self.check(&TokenKind::LParen)` loop in `call()` (`parser.rs:414-427`) with one that also handles `Dot`:

```rust
    fn call(&mut self) -> Result<Expr, CompileError> {
        let mut e = self.primary()?;
        loop {
            let (l, c) = self.here();
            if self.check(&TokenKind::LParen) {
                self.advance();
                let mut args = Vec::new();
                if !self.check(&TokenKind::RParen) {
                    loop {
                        args.push(self.expression()?);
                        if !self.matches(&TokenKind::Comma) { break; }
                    }
                }
                self.expect(&TokenKind::RParen, "')'")?;
                e = Expr::Call { callee: Box::new(e), args, line: l, col: c };
            } else if self.matches(&TokenKind::Dot) {
                let (field, _, _) = self.expect_ident("field or method name after '.'")?;
                if self.check(&TokenKind::LParen) {
                    self.advance();
                    let mut args = Vec::new();
                    if !self.check(&TokenKind::RParen) {
                        loop {
                            args.push(self.expression()?);
                            if !self.matches(&TokenKind::Comma) { break; }
                        }
                    }
                    self.expect(&TokenKind::RParen, "')'")?;
                    e = Expr::MethodCall { obj: Box::new(e), method: field, args, line: l, col: c };
                } else {
                    e = Expr::FieldGet { obj: Box::new(e), field, line: l, col: c };
                }
            } else {
                break;
            }
        }
        Ok(e)
    }
```

    fn object_slot_ptr(&self, base: inkwell::values::PointerValue<'ctx>, i: u64)
