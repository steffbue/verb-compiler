# Technology Stack

**Analysis Date:** 2026-07-21

## Languages

**Primary:**
- Rust 2021 edition - Compiler implementation, lexer, parser, AST, LLVM IR generation, JIT execution engine, LSP server

**Secondary:**
- C++ 17 - Runtime libraries for value management, I/O operations, and map data structures
- LLVM IR - Intermediate representation generated by Rust compiler for JIT and AOT compilation

## Runtime

**Environment:**
- LLVM 20.1 - JIT compiler and code generation backend
  - Installed via Homebrew: `/opt/homebrew/opt/llvm@20`
  - Configured in `.cargo/config.toml` via `LLVM_SYS_201_PREFIX`

**Package Manager:**
- Cargo 1.81+ (Rust package manager)
- Lockfile: `Cargo.lock` present

## Frameworks

**Core:**
- inkwell 0.9 - LLVM bindings for Rust
  - Feature: `llvm20-1` for LLVM 20.1 compatibility
  - Used in `src/codegen.rs` and `src/bin/verb-lsp.rs`

**Language Processing:**
- Custom lexer implementation (`src/lexer.rs`)
- Custom recursive descent parser (`src/parser.rs`)
- AST-based code generation (`src/ast.rs`)
- Custom formatter (`src/formatter.rs`)

**Build/Dev:**
- cc 1.0 - C/C++ compiler wrapper for runtime compilation
  - Compiles C++ runtime during build via `build.rs`
- zig - Cross-platform C toolchain (optional, for `--target` cross-compilation)

## Key Dependencies

**Critical:**
- inkwell 0.9 - LLVM IR generation and JIT compilation engine
  - Depends on: llvm-sys, bitflags, thiserror
- serde_json 1.0.151 - JSON serialization for LSP protocol
- cc 1.0 - Build-time C++ compilation

**Supporting:**
- libc 0.2.186 - System C library bindings
- thiserror 2.0.19 - Error type derivation
- anyhow 1.0.104 - Error context (indirect, via llvm-sys)
- proc-macro2, quote, syn - Macro support (indirect)

## Configuration

**Environment:**
- `.cargo/config.toml` - Sets `LLVM_SYS_201_PREFIX=/opt/homebrew/opt/llvm@20` for LLVM discovery
- No `.env` file - All configuration is compile-time or CLI-based

**Build:**
- `build.rs` - Compilation script that:
  - Compiles C++ runtime translation units (`runtime/verb_map.cpp`)
  - Links runtime symbols into the `verb` binary
  - Uses `cc::Build` with C++17 standard
  - Includes `runtime/verb.h` for C-ABI boundary definitions

**Linker Configuration:**
- For JIT (`verb run`): Links pre-compiled runtime objects
- For AOT host build (`verb build` without `--target`): Uses system `cc` compiler
- For cross-platform builds (`verb build --target <os>-<arch>`): Uses `zig cc` for cross-toolchain

## Platform Requirements

**Development:**
- Rust 2021 edition (stable or nightly)
- LLVM 20.1 with development headers
- C compiler (`cc`) - POSIX: gcc/clang, Windows: MSVC or mingw
- Optional: zig toolchain for cross-platform builds (supported targets: linux-x86_64, linux-arm64, macos-x86_64, macos-arm64, windows-x86_64, windows-arm64)

**Production:**
- No external runtime dependencies for generated binaries
- Generated binaries link libc and optional C++ stdlib for imports
- Windows targets require Visual C++ runtime or mingw dependencies depending on build toolchain

## Testing

- Fixtures directory: `tests/fixtures/` contains test programs and expected outputs
- E2E tests: Execute compiled binaries and verify output
- No external test framework (unittest, pytest, gtest)

**Artifacts:**
