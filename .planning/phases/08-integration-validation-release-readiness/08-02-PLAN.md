| Verb program → C++ FFI (mathlib) | Arguments/returns marshalled across the untyped VerbValue ABI |
| Verb program → filesystem (std io) | file_write/file_read cross the process/FS boundary |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-08-03 | Tampering | FFI boundary — c_add_int/c_* argument & return marshaling over VerbValue ABI | low | mitigate | Reuse the already-tested tests/fixtures/cpp/mathlib.cpp exports unchanged; the e2e output assertion catches any marshaling corruption |
| T-08-04 | Information Disclosure | std io file_write target path in the demo | low | mitigate | Path is a fixed literal ("verb_integration_demo.tmp"), never external/user input; file_write truncates (no stale append); test best-effort removes it |
| T-08-05 | Denial of Service | GC — a self-referential array/map would leak (GC-04) and break verb_gc_live=0 | low | mitigate | Example uses only acyclic structures; the zero-leaks assertion is the guard |

No package-manager installs (npm/pip/cargo add) occur in this plan; mathlib.cpp is an existing fixture, tests use std only — the supply-chain checkpoint (T-*-SC) is not applicable.
